Esta matéria cobre quatro conceitos que trabalham em conjunto para criar sistemas digitais funcionais e agradáveis. Entender a diferença — e a relação — entre eles é o ponto de partida para qualquer projeto de interface.

Aula 01 — Fundamentos
Experiência do Usuário
UX Design · Donald Norman

O termo User Experience foi cunhado por Donald Norman nos anos 1990, quando trabalhava na Apple. Ele sentiu que "usabilidade" e "interface" eram limitados demais — não capturavam tudo que envolve a interação de uma pessoa com um sistema.

"Eu inventei o termo porque achava que interface do usuário e usabilidade eram muito restritos. Eu queria cobrir todos os aspectos da experiência de uma pessoa com o sistema, incluindo design industrial, gráficos, a interface, a interação física e o manual."
Donald Norman — Designer e Cientista Cognitivo
"É o modo como você experiencia o mundo, é o modo como você experiencia sua vida, é o modo como você experiencia um serviço."
Donald Norman — O Design do Dia a Dia

UX é uma experiência contínua — não é um momento único de uso, mas toda a relação construída entre o usuário e o produto ao longo do tempo.

nota

"O Design do Dia a Dia" de Donald Norman explica como objetos cotidianos (portas, torneiras, fogões) podem ter design bom ou terrível, e o que isso nos ensina sobre interfaces digitais. Leitura altamente recomendada.

referência

Em UX, não existe um padrão "tamanho único" para uma determinada situação. Cada contexto, público e objetivo exige uma abordagem diferente. — Mike Atherton, UX is UI, 2015.

Pesquisa & Foco no Usuário
Entendendo quem usa

Antes de projetar qualquer coisa, UX exige entender profundamente quem é o usuário. Isso envolve quatro dimensões fundamentais:

CaracterísticasQuem é essa pessoa? Idade, nível de instrução, classe social, familiaridade com tecnologia. Essas informações definem o vocabulário, a complexidade e o tom da interface.
NecessidadesO que essa pessoa precisa fazer? Contratar um serviço, comprar um produto, acessar uma informação. As necessidades definem as funcionalidades essenciais.
DesejosO que essa pessoa quer além do necessário? Sentir-se valorizada, economizar tempo. Os desejos guiam as decisões de encantamento e diferenciação.
DificuldadesO que pode impedir essa pessoa? Limitações motoras, pouca familiaridade com tecnologia, conexão lenta. As dificuldades revelam onde a interface precisa ser mais cuidadosa.

Para entender necessidades e desejos, usamos dois tipos principais de pesquisa:

Quantitativo

Questionários

Google Forms, SurveyMonkey etc. Coletam dados de muitas pessoas rapidamente. Indicado para entender padrões e frequências em escala.

Qualitativo

Entrevistas

Conversas individuais gravadas em áudio/vídeo que revelam motivações profundas e comportamentos que questionários jamais capturam.

dica

Na prática, combine os dois métodos: questionários para o cenário geral, entrevistas para aprofundar os pontos críticos. Essa combinação se chama triangulação de dados.

Arquitetura da Informação
User Flow & Estrutura

A AI usa os dados de UX para criar estrutura. Aqui decide-se como o conteúdo será organizado e como o usuário vai navegar entre telas. Pense na AI como a planta baixa de um prédio — antes de erguê-lo, você define onde ficam os cômodos e as portas.

O resultado desta etapa é o User Flow — um diagrama que mostra todos os caminhos possíveis que o usuário pode percorrer no sistema.

01

Define o escopo

Ao mapear todas as telas e conexões, você visualiza se o projeto é pequeno ou grande — evitando surpresas no meio do desenvolvimento.

02

Facilita a comunicação

Um fluxo visual é muito mais fácil de apresentar para clientes e stakeholders do que uma lista de funcionalidades em texto.

03

Revela problemas cedo

É muito mais barato corrigir um fluxo confuso no diagrama do que depois que telas foram desenhadas ou código foi escrito.

ferramenta

O professor usa Figma + FigJam. O FigJam é o quadro branco colaborativo do Figma — gratuito para uso básico, ideal para criar user flows com post-its, setas e diagramas em equipe.

Wireframe & Prototipagem
Do rascunho ao produto

O wireframe é um esboço estrutural da interface — sem cores, fontes elaboradas ou imagens reais. O objetivo é definir o layout, a hierarquia de informação e as funcionalidades de cada tela, sem se distrair com estética.

Nível 01
Baixa Fidelidade
Rabiscos no papel. Rápidos, descartáveis, feitos para explorar ideias.
Nível 02
Média Fidelidade
Feito no Figma. Já tem estrutura real, conexões entre telas e interações básicas.
Nível 03
Alta Fidelidade
O protótipo final. Com cores, imagens e animações — praticamente o produto real.

Interface do Usuário
UI Design

Enquanto UX se preocupa com o que o sistema precisa fazer, a UI se preocupa com a aparência e o controle visual — tudo que o usuário vê na tela: hierarquia visual, cores, tipografia, botões, ícones, espaçamento e responsividade.

"A UI cria caminhos para efetivar a comunicação entre usuário e sistema."
Prof. Flávio Waltz
Usabilidade
ISO/IEC 9126 · 5 Subcaracterísticas

A norma ISO/IEC 9126 define usabilidade como a "capacidade do produto de software de ser compreendido, aprendido, operado e atraente ao usuário" nas condições de uso especificadas.

01

Inteligibilidade

O usuário consegue entender para que serve o software antes mesmo de usá-lo? Ícones claros, títulos descritivos e onboarding eficaz são parte disso.

02

Apreensibilidade

Quão rápido o usuário aprende a usar o sistema? Isso se constrói com padrões familiares, affordances visuais claros e hierarquia bem definida.

03

Operacionalidade

O usuário consegue controlar o sistema com facilidade? Feedbacks claros, ação de desfazer e confirmação antes de ações destrutivas.

04

Atratividade

O sistema é visualmente agradável? Interfaces bonitas geram mais confiança e reduzem a percepção de dificuldade — o chamado efeito estético-usabilidade.

05

Conformidade

O sistema segue normas e guias de estilo como Material Design (Google) ou HIG (Apple)? Padrões conhecidos aceleram o aprendizado do usuário.

Design de Interação
IxD · Artefatos Interativos

O IxD estuda o comportamento de artefatos interativos — produtos e sistemas que geram resposta quando acionados. O objetivo é criar interações que não exijam esforço cognitivo e que sejam significativas.

Feedback imediato

O sistema deve sempre confirmar que recebeu a ação. Um botão que muda de cor, um loader, um "enviado" — todos reduzem ansiedade.

Sem esforço

Interações bem desenhadas parecem naturais. Microcopy claro, gestos intuitivos, hierarquia óbvia e consistência entre telas.

Significativas

Cada interação deve ter propósito. Animações decorativas distraem; animações que guiam a atenção ou comunicam mudança de estado são valiosas.

conexão

IxD está no centro do diagrama de Dan Saffer (2009) — o núcleo onde se intersectam o design visual, os fatores humanos e a interação humano-computador.

Aula 02 — Heurísticas de Nielsen
Jakob Nielsen
Um dos maiores estudiosos da usabilidade

Jakob Nielsen é cofundador do Nielsen Norman Group e um dos pesquisadores de usabilidade mais influentes do mundo. Ele desenvolveu as 10 Heurísticas de Usabilidade, um conjunto de princípios que guiam a avaliação e o design de interfaces.

"As 10 heurísticas em si permaneceram relevantes e inalteradas desde 1994."
Jakob Nielsen
o que é

Uma avaliação heurística é um método de inspeção de usabilidade onde especialistas avaliam uma interface com base em princípios reconhecidos — sem precisar envolver usuários reais. É rápida, barata e eficaz.

1. Visibilidade do Status do Sistema
Heurística de Nielsen · Feedback

Quando os usuários conhecem o status atual do sistema, eles aprendem o resultado de suas interações anteriores e determinam as próximas etapas. Interações previsíveis criam confiança no produto e na marca.

→

Após ação do usuário

Ao enviar um formulário, exibir confirmação. Ao clicar em "adicionar ao carrinho", mostrar badge com número de itens atualizado.

→

Status persistente

Barras de progresso durante carregamentos, indicadores de etapa em formulários multi-passo, status "online/offline" em apps de mensagem.

exemplo prático

O Gmail mostra "Enviando..." e depois "Mensagem enviada" com opção de desfazer — visibilidade de status perfeita.

2. Correspondência com o Mundo Real
Heurística de Nielsen · Linguagem

O design deve falar a linguagem dos usuários. Use palavras, frases e conceitos familiares, em vez de jargão interno. Siga as convenções do mundo real, fazendo que as informações apareçam em ordem natural e lógica.

Bom exemplo

Carrinho de compras

Metáfora do mundo físico. O usuário entende imediatamente sem precisar aprender.

Atenção

Adaptação Regional

Formatos de data mudam por país: DD/MM/AAAA no Brasil vs MM/DD/YYYY nos EUA. Adapte sempre ao contexto do seu usuário.

cuidado com excessos

Um ícone de disquete para "salvar" faz sentido para quem conhece disquetes — mas para jovens usuários, é apenas um símbolo aprendido. Evite metáforas que perderam a referência.

3. Controle e Liberdade do Usuário
Heurística de Nielsen · Saída de emergência

Os usuários geralmente executam ações por engano. Eles precisam de uma "saída de emergência" claramente marcada para deixar a ação indesejada sem ter que passar por um processo prolongado.

→

Desfazer e refazer

Ctrl+Z é o exemplo mais claro. Em interfaces digitais, qualquer ação destrutiva deve ser reversível sempre que possível.

→

Botões de cancelamento

Todo modal ou processo multi-etapa deve ter uma forma clara de sair. O usuário não pode se sentir "preso" em um fluxo.

→

Confirmação antes de ações críticas

"Tem certeza que deseja excluir?" — esse diálogo dá ao usuário a chance de reconsiderar antes de uma ação irreversível.

4. Consistência e Padrões
Heurística de Nielsen · Previsibilidade

Os usuários não devem se perguntar se palavras, situações ou ações diferentes significam a mesma coisa. Siga as convenções da plataforma e do setor.

Consistência Interna

Dentro do próprio sistema

Se um botão vermelho significa "excluir" em uma tela, deve significar o mesmo em todas as outras. Padrões internos reduzem a curva de aprendizado.

Consistência Externa

Com outros sistemas

Seguir padrões de mercado (Material Design, iOS HIG) aproveita o conhecimento que o usuário já tem de outros apps.

analogia

Os balcões de check-in geralmente ficam na entrada de hotéis. Essa consistência atende às expectativas dos clientes — eles nem precisam procurar.

5. Prevenção de Erros
Heurística de Nielsen · Proativo

Boas mensagens de erro são importantes, mas os melhores designs evitam cuidadosamente a ocorrência de problemas. Elimine as condições propensas a erros ou verifique-as e apresente opções de confirmação antes de ações comprometedoras.

→

Limitar possibilidades

Em vez de deixar o usuário digitar qualquer coisa e depois informar um erro, use campos com formato definido, seletores de data, máscaras de input.

→

Confirmação para ações críticas

"Você está excluindo 3 arquivos permanentemente. Continuar?" — impede erros irreversíveis por descuido.

→

Guardrails

Como grades de proteção em estradas de montanha: não impedem de dirigir, mas protegem de quedas. Na interface: alertas de campo vazio antes de enviar.

6. Reconhecimento ao Invés de Lembrança
Heurística de Nielsen · Carga cognitiva

Minimize a carga de memória do usuário tornando visíveis elementos, ações e opções. O usuário não deve ter que lembrar informações de uma parte da interface para outra.

Reconhecer

"Lisboa é a capital de Portugal?"

O usuário apenas confirma ou nega uma opção apresentada. Muito menos esforço mental.

Lembrar

"Qual é a capital de Portugal?"

O usuário precisa recuperar a informação da memória sem ajuda. Muito mais difícil — evite isso na sua interface.

exemplo prático

Menus de navegação visíveis são melhores que menus escondidos. O usuário não deve ter que lembrar onde está cada funcionalidade.

7. Flexibilidade e Eficiência de Uso
Heurística de Nielsen · Atalhos

Atalhos podem acelerar a interação para o usuário experiente sem prejudicar o novato. O design deve atender a ambos os perfis e permitir que usuários personalizem ações frequentes.

→

Atalhos de teclado

Ctrl+S, Ctrl+Z, Ctrl+C — usuários avançados usam sem nem pensar. Iniciantes aprendem gradualmente.

→

Múltiplos caminhos para a mesma ação

Arrastar um arquivo para fazer upload e clicar em "selecionar arquivo" fazem a mesma coisa. O usuário escolhe o que prefere.

→

Funcionalidades "aceleradoras"

Favoritos, histórico de busca, templates, macros — formas de o sistema aprender com o usuário e acelerar tarefas repetidas.

8. Design Estético e Minimalista
Heurística de Nielsen · Hierarquia visual

As interfaces não devem conter informações irrelevantes. Cada unidade extra de informação compete com as unidades relevantes e diminui sua visibilidade relativa.

"Todas as informações são pertinentes. As imagens estão comunicando e não decorando."
Jakob Nielsen — Heurística 8
analogia

Um bule ornamentado pode ter elementos decorativos que interferem na usabilidade. Na interface, decoração sem função é ruído visual que prejudica o uso.

importante

Minimalismo não significa "sem estilo" — significa que cada elemento tem um propósito. O que se elimina é o supérfluo, não o expressivo.

9. Reconhecimento, Diagnóstico e Recuperação de Erros
Heurística de Nielsen · Mensagens de erro

As mensagens de erro devem ser expressas em linguagem simples (sem códigos de erro), indicar com precisão o problema e sugerir uma solução de forma construtiva.

✗

Errado: "Erro 404"

O usuário não sabe o que isso significa e não sabe o que fazer. Não ajuda em nada.

✓

Certo: "Página não encontrada — voltar ao início?"

Informa o problema em linguagem humana e oferece uma saída clara. O usuário pode se recuperar sem frustração.

→

Use cor + texto

Combine indicador visual com texto descritivo para informar o erro. Nunca use apenas cor — acessibilidade importa.

10. Ajuda e Documentação
Heurística de Nielsen · Suporte

É melhor que o sistema não precise de nenhuma explicação adicional. No entanto, pode ser necessário fornecer documentação para ajudar os usuários a entender como concluir suas tarefas.

→

Contextualizado

A ajuda deve ser focada na tarefa atual do usuário, não genérica. Quiosques de aeroporto são facilmente identificáveis e resolvem problemas no local.

→

Sugestões em tempo real

Tooltips, placeholders explicativos em campos, textos de ajuda abaixo de inputs — documentação integrada ao fluxo.

→

Guias passo a passo

Para tarefas complexas, ofereça um wizard ou tutorial que leva o usuário pela tarefa sem sobrecarga cognitiva.

Aula 03 — Elementos da Experiência do Usuário
Jesse James Garrett
The Elements of User Experience, 2011

Jesse James Garrett desenvolveu um modelo que organiza o processo de UX em cinco planos interdependentes, do mais abstrato ao mais concreto. Cada plano depende das decisões tomadas no plano anterior.

Plano Estratégico
Mais abstrato
Define os objetivos do produto e as necessidades do usuário. É a fundação de tudo.
Plano de Escopo
Funcionalidades
Define quais features e conteúdos o produto terá. O que será construído.
Plano de Estrutura
Arquitetura
Como o conteúdo e as funcionalidades se organizam. User flow e arquitetura da informação.
Plano de Esqueleto
Layout
Onde cada elemento fica na tela. Wireframes e organização visual das informações.
Plano de Superfície
Mais concreto
O que o usuário vê: cores, tipografia, imagens, contraste e padrões visuais.
Plano Estratégico
Objetivos do produto & Necessidades do usuário

O Plano Estratégico responde às perguntas mais fundamentais: para que serve este produto? e para quem? Ele deve ser o início do processo de design, mas não deve ser absolutamente imutável.

Objetivos do produtoInformar, ensinar, vender, oferecer um serviço... O que o produto precisa realizar para o negócio?
Necessidades do usuárioO que o usuário quer conquistar ao usar o produto? Não o que ele diz que quer, mas o que realmente precisa.

O público pode ser extremamente variado. Por isso, é interessante segmentar os usuários por suas características:

Demográfico

Dados objetivos

Idade, raça, sexo, renda, localização. O perfil mensurável do usuário.

Psicográfico

VALS

Valores, atitudes e estilo de vida. O que motiva e influencia as decisões do usuário.

Tecnológico

Familiaridade digital

Tempo e nível de uso de tecnologia. Determina o nível de complexidade adequado.

exemplo

Vender utensílios de cozinha para cozinheiros amadores é diferente de vendê-los para cozinheiros profissionais. Mesmo produto, públicos distintos, interfaces completamente diferentes.

Plano de Escopo
Features, Conteúdo & Requisitos

O Plano de Escopo define quais funcionalidades e conteúdos o produto terá. Pesquisas (internas e externas) ajudam a determinar o que construir — e o que não construir.

01

Defina requisitos com precisão

"Os vídeos mais populares serão destacados" é vago. Sempre especifique: "Os vídeos com maior número de visualizações na última semana aparecerão no topo da lista."

02

Requisitos têm limites de cronograma

Funcionalidades que não cabem no sprint atual podem formar a base para o próximo ciclo de desenvolvimento. Priorização é essencial.

03

Considere o hardware disponível

Processador, GPS, giroscópio, câmera — as features do produto dependem do que o dispositivo do usuário suporta.

04

Defina o conteúdo

Texto, imagem, áudio e vídeo. Determine também tamanho de texto, dimensões de imagens, formatos de arquivo — tudo que impacta performance e experiência.

Plano de Estrutura
Arquitetura da Informação & Modelos Conceituais

O Plano de Estrutura une o design de interação e a arquitetura da informação para definir padrões e sequências apresentadas ao usuário.

É importante utilizar modelos conceituais — linguagens já conhecidas pelo usuário. Um carrinho de compras é um modelo conceitual: o usuário sabe o que é sem precisar aprender.

cuidado

Cuidado com o excesso de metáforas físicas. Tentar replicar uma prateleira de livraria 3D em um e-commerce parece criativo, mas dificulta a navegação. A grade simples de produtos é mais eficiente.

A Arquitetura da Informação pode ter quatro tipos de estrutura:

Mais comum

Hierárquica

Nós pais (informações amplas) e nós filhos (informações específicas). Árvore de categorias de e-commerces, por exemplo.

Liberdade

Matriz tridimensional

Diferentes recursos para usuários com diferentes necessidades. Filtros de busca são o melhor exemplo.

Exploração

Orgânica

Não segue padrões consistentes. Usada em sites educativos ou de entretenimento que incentivam a exploração livre.

Linear

Sequencial

Característica de mídias offline (livros, vídeos, apresentações). O usuário segue uma ordem pré-definida.

Aula 04 — Esqueleto e Superfície
Plano de Esqueleto
Refinando para a interface concreta

O Plano de Esqueleto é a hora de refinar os planos anteriores e torná-los mais concretos, focando nos aspectos da interface. O objetivo é garantir boa usabilidade através de escolhas conscientes de layout e componentes.

Componentes de Interface
Os elementos com que o usuário interage

Cada componente de UI tem uma função específica — usar o componente errado para um contexto é um erro clássico de design. Veja as principais diferenças:

UI 01

Checkbox

Permitem que os usuários selecionem uma ou mais opções de forma independente. Também serve para ativar/desativar um item.

✓ Use quando: selecionar múltiplas opções de uma lista, apresentar sub-seleções, ativar/desativar um recurso no desktop.
UI 02

Botão de Rádio (Radio Button)

Permite que apenas uma opção seja selecionada em uma lista visível. Todas as opções ficam expostas ao mesmo tempo.

✓ Use quando: selecionar exatamente uma opção de uma lista. ✗ Não use Checkbox no lugar de Radio quando só uma opção pode ser escolhida.
UI 03

Menu Dropdown

Permite que os usuários façam uma seleção entre várias opções sem ocupar espaço visível na tela. As opções ficam ocultas até o clique.

✓ Use quando há muitas opções e espaço é limitado. Se há poucas opções e espaço suficiente, prefira Radio Buttons visíveis.
UI 04

Call to Action (CTA)

Botão com texto no modo imperativo que dá uma ordem ou faz um pedido claro ao usuário.

Do menos eficaz ao mais eficaz: "Inscreva-se" → "Inscreva-se hoje" → "Inscreva-se de graça". Especificidade e urgência aumentam conversão.
Sistemas de Navegação
Como o usuário se move pelo sistema

Um sistema de navegação deve: (1) fornecer um meio de ir de um ponto a outro no site e (2) comunicar a relação entre os elementos contidos no sistema. Não basta uma lista de links — é preciso mostrar hierarquia e contexto.

Tipo 01

Global

Disponibiliza pontos de acesso "chave" para tornar a navegação eficiente. Ex: barra de navegação principal no topo do site.

Tipo 02

Local

Dá acesso a conteúdo "próximo" ou semelhante ao que o usuário está vendo. Ex: sidebar com posts relacionados.

Tipo 03

Suplementar

Permite mudar o foco da navegação sem retornar ao início. Ex: filtros laterais que mudam o conteúdo sem recarregar a página.

Tipo 04

Contextual

Fornece links para áreas "distantes" mas relacionadas ao contexto atual. Ex: hiperlinks dentro do texto que levam a outros artigos.

Tipo 05

Cortesia

Acesso a informações pertinentes mas raramente procuradas: contato, endereço, FAQ, política de privacidade — geralmente no rodapé.

Tipo 06

Design de Informação

Como apresentar dados de forma que o usuário compreenda facilmente. Gráficos, tabelas e visualizações comunicam mais que texto bruto.

Wayfinding — Orientação
Onde estou? Para onde posso ir?

Wayfinding é a prática de ajudar pessoas a entenderem onde estão e para onde podem ir. A ideia é inspirada nos elementos de sinalização presentes no espaço físico: metrô, shoppings, estradas.

→

Breadcrumbs (Migalhas de pão)

Início > Produtos > Eletrônicos > Smartphones — indica o caminho percorrido e permite voltar a qualquer ponto.

→

Navegação local visível

A estrutura de seções do site deve estar sempre acessível. O usuário pode ter entrado por uma URL direta e não saber onde está.

URLs descritivas

Plano de Superfície
Eyetracking, Contraste & Padrões Visuais

O Plano de Superfície é o mais visível — tudo que o usuário enxerga. Não é só sobre estética: é sobre como o olho humano percebe e processa a interface.

S 01

Banner Blindness

Pesquisas de eyetracking mostram que as pessoas evitam elementos que percebem como anúncios — mesmo que não sejam. Banners em locais típicos de publicidade são ignorados automaticamente.

S 02

Flat UI e Ghost Buttons

Elementos muito "flat" exigem mais esforço do usuário para identificar o que é clicável. Ghost buttons reduzem a taxa de clique em comparação com botões preenchidos.

S 03

Contraste Visual

O contraste é vital para guiar o olhar do usuário para aspectos essenciais da interface. Sem contraste, tudo compete pela atenção. Com contraste excessivo, a interface fica poluída.

01

Crie interfaces previsíveis

Padrões familiares aceleram o aprendizado. O usuário não deve ser surpreendido por comportamentos inesperados.

02

Crie hierarquias visuais

O olho do usuário deve ser guiado naturalmente pelo que é mais importante. Tamanho, peso, cor e posição criam hierarquia.

03

Esconda o que não está sendo usado

Elementos de interface irrelevantes ao contexto atual devem ficar ocultos — reduzem ruído e aumentam o foco.

Aula 05 — Testes de Usabilidade
Testes de Usabilidade
Validando com usuários reais

Um teste de usabilidade é a etapa onde usuários reais interagem com o sistema enquanto pesquisadores observam e coletam dados. Existem pilares fundamentais para conduzir um bom teste:

T 01

Escolha participantes representativos

Os participantes devem ser similares ao público-alvo real, não devem ter muito conhecimento de UX ou Web Design (isso enviesa os resultados) e não devem ser participantes "internos" da equipe.

T 02

Dê tarefas realistas

Em vez de "Entre no site, busque um produto, faça login e compre", use contextos: "Imagine que você quer presentear um amigo e tem R$100 disponíveis". O participante age naturalmente dentro de um cenário verossímil.

T 03

Fale o mínimo possível

O moderador deve não enviesar o participante. Observe e anote, mas não conduza. Deixe o participante confortável — o que está sendo testado é o sistema, não a pessoa.

T 04

Priorize o que testar

Primeiro: o que pode ser modificado. Segundo: pontos nunca antes testados ou que oferecem risco. Terceiro: pontos que impactam o usuário ou o negócio (como o fluxo de login ou de compra).

Qualitativo vs. Quantitativo
Dois tipos de dados, dois objetivos

Os dados coletados em testes de usabilidade são de dois tipos fundamentais, cada um com características e aplicações distintas:

Dimensão Qualitativo Quantitativo
O que é Achados observacionais: o que o usuário faz, diz e sente durante o teste Métricas numéricas: taxas de conclusão, tempo de tarefa, número de erros
Para que serve Avaliação direta da usabilidade — identifica exatamente quais elementos causam problema Avaliação indireta — mede se algo está funcionando, mas não explica por quê
Participantes ~5 participantes são suficientes para encontrar a maioria dos problemas ~30 participantes para ter dados estatisticamente significativos
Quando usar Para avaliar partes ou processos específicos do projeto. Mais barato e rápido. Para avaliar todo o sistema. Mais caro, exige mais tempo e controle rigoroso.
Limitação Menos generalizável — o que 5 pessoas sentem pode não representar o todo "Saber que 40% falham não diz por quê falharam ou como corrigir." — Nielsen
combinação ideal

Estudos qualitativos costumam ser usados para avaliar partes específicas do projeto. Estudos quantitativos tendem a ser usados para a avaliação do sistema como um todo. Na prática ideal, os dois se complementam.

Metodologia dos Testes
Como conduzir bem um teste

Além do tipo de dado coletado, a qualidade do teste depende da metodologia aplicada. Dois critérios são fundamentais para qualquer estudo:

Critério

Validade Externa

Os participantes são representativos do público-alvo e as condições refletem o uso real. Ex: testar um site mobile em simulador de desktop carece de validade externa.

Critério

Validade Interna

A configuração não favorece nenhuma condição. Ex: se o design A é testado de manhã e B à tarde, a fadiga pode afetar os resultados do B.

01

Randomize as tarefas

A ordem das tarefas não deve influenciar os resultados. Principalmente em testes quantitativos, randomizar garante que nenhum viés de ordem contamine os dados.

02

Protocolo "pensar em voz alta"

Padrão em estudos qualitativos — o participante narra o que está pensando enquanto usa o sistema. Em estudos quantitativos, esse protocolo pode interferir nos dados de tempo.

03

Controle rigoroso no quantitativo

Não é possível executar parte dos testes à distância e parte presencialmente. Tudo deve ser idêntico para garantir comparabilidade.

04

Tarefas com resposta definida (quantitativo)

"Encontre o telefone e endereço de João Silva" — do ponto de vista quantitativo, ruim: e se só o telefone for encontrado? A métrica fica ambígua.

05

Flexibilidade no qualitativo

No qualitativo, não há problema em ajustar as condições entre sessões. Se uma tarefa não está gerando os insights necessários, reescreva antes do próximo participante.
